# `/go-build` — Go 빌드/정적분석 에러를 “수술적 최소 변경”으로 복구하는 커맨드

`/go-build`는 Go 프로젝트에서 `go build` 실패, `go vet` 경고, 린터 이슈 등을 **하나씩 분리해서 최소 수정으로 해결**하도록 돕는 슬래시 커맨드입니다.  
내부적으로는 **`go-build-resolver` 에이전트를 호출**해 “한 번에 하나씩 고치고 바로 재검증”하는 방식으로 build green을 복구합니다.

# 한눈에 보기(요약)

### 역할

- Go 진단 커맨드를 실행(`go build`, `go vet`, `staticcheck` 등)
- 에러를 파일별로 묶고 심각도/의존성 순서로 정렬
- **1개 이슈씩** 최소 diff로 수정하고 매번 다시 실행해 검증
- 최종적으로 “무엇을 고쳤고 무엇이 남았는지” 요약 리포트 생성

### 언제 쓰나

- `go build ./...`가 실패할 때
- `go vet ./...`에서 이슈가 나올 때
- `golangci-lint run` 경고가 쌓였을 때
- `go mod` 의존성/모듈 상태가 깨졌을 때
- 변경사항을 받아온 뒤(merge/pull) 빌드가 깨졌을 때

### 출력(결과물)

- 초기 진단 결과(에러 수/위치)
- 각 이슈별 수정 내용(파일/원인/해결)
- 재검증 결과(남은 에러 수 감소 과정)
- 최종 요약(수정 건수, 수정 파일 수, 잔여 이슈)

# 어떻게 쓰나(1분 사용법)

1. 채팅에 `/go-build`를 입력합니다.
2. 에이전트가 진단 커맨드를 실행해 실패 원인을 수집합니다.
3. 에러를 “하나씩” 최소 변경으로 고치고, 매 수정마다 `go build`로 확인합니다.
4. 빌드가 성공하면 `go vet`/테스트 등 후속 검증을 이어갑니다.

# 이 커맨드가 하는 일(진행 흐름)

### 1) 진단(먼저 실행하는 커맨드)

```bash
go build ./...
go vet ./...

# 설치되어 있으면 추가 진단
staticcheck ./...
golangci-lint run

# 모듈 상태 점검/정리
go mod verify
go mod tidy -v
```

### 2) 에러 정리 및 점진적 수정

- 에러를 파일별로 그룹핑하고, 심각도/의존성 순으로 처리
- 한 번에 하나씩 해결 → 바로 재검증 → 다음 이슈로 이동
- 우선순위 권장:
  - **빌드 에러**(컴파일이 최우선)
  - **vet 경고**
  - **lint 경고**

### 3) 중단 조건(리포트하고 멈춤)

- 같은 에러가 **3번 시도해도 해결되지 않음**
- 수정했더니 **에러가 더 늘어남**
- 해결에 **아키텍처 변경**이 필요
- 외부 의존성/환경 문제로 **코드만으로 해결 불가**

# 실무 팁 / 주의사항(중요)

- 목표는 “코드 품질 개선”이 아니라 **빌드 복구**입니다. 큰 리팩터링은 분리하세요.
- `go mod tidy`는 변경폭이 커질 수 있으니, 원인이 모듈 문제로 확실할 때 실행하는 게 안전합니다.

# Quick Commands(실행 예)

```bash
go build ./...
go vet ./...
go test ./...
staticcheck ./...
golangci-lint run
go mod verify
go mod tidy -v
```
