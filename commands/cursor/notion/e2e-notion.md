# `/e2e` — Playwright로 “사용자 여정(E2E) 테스트”를 생성·실행·아티팩트까지 남기는 커맨드

`/e2e`는 **Playwright 기반의 End-to-End 테스트를 “사용자 흐름(journey)” 단위로 설계/생성하고, 실제 브라우저에서 실행한 뒤, 실패 증거(스크린샷/비디오/트레이스)와 리포트(HTML/JUnit)까지 남기도록** 돕는 슬래시 커맨드입니다.  
내부적으로는 **`e2e-runner` 에이전트를 호출**해 테스트 생성/유지보수/실행을 한 번에 처리합니다.

# 한눈에 보기(요약)

### 역할

- 사용자 흐름 분석
- 시나리오 도출
- (POM 기반) Playwright 테스트 생성/유지
- 멀티 브라우저 실행
- 실패 아티팩트/리포트 생성
- 플래키 탐지/격리 제안

### 언제 쓰나

- 로그인/결제/거래 같은 **핵심 사용자 여정**이 실제로 끝까지 되는지 확인할 때
- 프론트-백엔드가 엮이는 **멀티스텝 플로우 회귀 테스트**가 필요할 때
- 릴리즈/배포 전 **통합 안정성 검증**이 필요할 때

### 출력(결과물)

- **테스트 코드**(보통 Page Object Model 패턴)
- **실행 결과 요약**
- **HTML 리포트**, **JUnit XML(CI용)**
- (실패 시) **스크린샷/비디오/트레이스/네트워크·콘솔 로그**
- **플래키(불안정) 테스트 탐지** 및 `test.fixme()` 등 격리 권고

# 어떻게 쓰나(1분 사용법)

1. 채팅에 `/e2e`를 입력하고 **검증하고 싶은 사용자 흐름을 자연어로** 적습니다.
   - 예: `/e2e 마켓 검색 → 결과 클릭 → 상세 진입 흐름 테스트해줘`
2. `e2e-runner`가 흐름을 분석해 **테스트 시나리오/커버리지**를 먼저 잡고,
3. **테스트 코드 생성 + 실제 실행**을 수행한 다음,
4. **리포트/아티팩트**와 함께 결과를 정리해줍니다.

# 이 커맨드가 하는 일(진행 흐름)

`/e2e`는 대략 아래 순서로 동작합니다.

- **사용자 흐름 분석**: 어떤 페이지/상태/전이(네비게이션)가 핵심인지 정리
- **시나리오 식별**: “정상 플로우 + 대표적인 예외(빈 결과 등)”를 테스트 케이스로 구성
- **테스트 생성(유지보수 친화)**:
  - **Page Object Model(POM)** 형태로 페이지 동작을 추상화
  - 선택자는 **`data-testid`** 기반을 권장(바이틀한 CSS 클래스 회피)
  - 임의 `sleep` 대신 **API 응답/로드 상태 대기**를 우선
- **멀티 브라우저 실행**: 기본적으로 Chromium/Firefox/WebKit 등에서 실행(구성은 `playwright.config.ts`)
- **아티팩트/리포트 생성**:
  - 모든 실행: **HTML Report, JUnit XML**
  - 실패 시: **스크린샷/비디오/트레이스 + 네트워크/콘솔 로그**
- **플래키 탐지**: 간헐 실패 패턴을 요약하고, **명시적 wait/timeout 조정/레이스 조건 점검** 등 개선안을 제시하며 필요 시 **격리(`test.fixme()`)**를 권장

# 실무 팁 / 주의사항(중요)

- **E2E는 “모든 엣지 케이스”가 아니라 “핵심 사용자 여정” 중심**으로 두고, 나머지는 `/tdd`로 유닛/통합 테스트에 맡기는 게 효율적입니다.
- **프로덕션 대상 실행 금지(특히 금전/거래)**:
  - 돈/거래 관련 E2E는 **testnet/staging에서만** 수행
  - 환경변수로 **production에서 자동 skip** 같은 가드(`test.skip(...)`)를 두는 패턴을 권장
- **다른 커맨드와의 연결**
  - `/plan`: “어떤 여정이 크리티컬인지”를 먼저 확정
  - `/tdd`: 빠르고 촘촘한 단위 테스트
  - `/e2e`: 최종 사용자 관점 통합 검증
  - `/code-review`: 테스트 품질/안정성 점검

# Quick Commands(실행 예)

```bash
# 전체 E2E 실행
npx playwright test

# 특정 테스트만 실행
npx playwright test tests/e2e/...spec.ts

# 브라우저를 띄워서 실행(headed)
npx playwright test --headed

# 디버그 모드
npx playwright test --debug

# 리포트 보기
npx playwright show-report
```
