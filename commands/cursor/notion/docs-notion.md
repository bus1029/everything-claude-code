# `/docs` — 라이브러리·프레임워크의 최신 문서를 Context7로 찾아 요약하는 커맨드

`/docs`는 라이브러리, 프레임워크, API에 대해 **최신 공식 문서 기반 답변**을 빠르게 얻기 위한 슬래시 커맨드입니다.  
핵심은 모델의 오래된 기억에 기대기보다, **Context7로 실제 문서를 조회한 뒤 필요한 코드 예시까지 함께 요약**하는 데 있습니다.

# 한눈에 보기(요약)

### 역할

- 라이브러리 이름과 질문을 받아 최신 문서 검색
- `resolve-library-id`로 대상 문서를 정확히 식별
- `query-docs`로 관련 문서를 조회
- 답변과 함께 실무에 바로 쓸 수 있는 코드 예시 정리

### 언제 쓰나

- 프레임워크 설정법을 최신 기준으로 확인하고 싶을 때
- 특정 API의 사용법이나 옵션 차이를 빠르게 알고 싶을 때
- 버전 의존적인 문서를 “기억”이 아니라 실제 문서로 확인하고 싶을 때

### 출력(결과물)

- 조회한 라이브러리 식별 결과
- 질문에 대한 최신 문서 기반 요약 답변
- 필요 시 함께 제시되는 코드 스니펫
- Context7 사용 불가 시 best-effort 답변과 주의 문구

# 어떻게 쓰나(1분 사용법)

1. `/docs [library name] [question]` 형태로 입력합니다.
2. 공백이 있는 이름이나 질문은 따옴표로 감쌉니다.
3. 라이브러리명이나 질문이 빠지면, 에이전트가 부족한 정보를 다시 묻습니다.

예:

`/docs "Next.js" "How do I configure middleware?"`

# 이 커맨드가 하는 일(진행 흐름)

### 1) 라이브러리 식별

- 입력받은 라이브러리명과 질문을 바탕으로 `resolve-library-id`를 호출합니다.
- 예: `Next.js` → `/vercel/next.js`

### 2) 문서 조회

- 식별된 라이브러리 ID와 질문으로 `query-docs`를 호출합니다.
- 질문 범위에 맞는 최신 문서 조각을 가져옵니다.

### 3) 답변 요약

- 문서 내용을 바탕으로 핵심만 짧게 요약합니다.
- 필요한 경우 설정 예시, 코드 스니펫, 버전 관련 주의사항을 함께 정리합니다.

### 4) 예외 처리

- Context7을 사용할 수 없으면 그 사실을 먼저 알립니다.
- 그 경우에는 모델 지식을 바탕으로 답하되, 최신성과 정확도에 한계가 있을 수 있음을 함께 표시합니다.

# 실무 팁 / 주의사항(중요)

- 질문은 넓게보다 **구체적으로** 적는 편이 더 정확한 답을 얻기 쉽습니다
- 버전 차이가 큰 라이브러리는 질문에 버전 정보를 함께 넣는 게 좋습니다
- “설치법”보다 “어떤 환경에서 무엇을 하려는지”까지 적으면 더 실무적인 답이 나옵니다

# Quick Commands(실행 예)

```text
/docs React useEffect cleanup
/docs "Next.js" "How do I configure middleware?"
/docs Prisma transactions
/docs "Supabase" "Auth methods"
```
