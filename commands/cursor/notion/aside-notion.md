# `/aside` — 현재 작업을 끊지 않고 “짧은 옆질문”에 답한 뒤 바로 복귀하는 커맨드

`/aside`는 작업 도중 생긴 짧은 질문에 **맥락을 잃지 않고 즉시 답한 뒤**, 원래 하던 작업으로 다시 돌아가기 위한 슬래시 커맨드입니다.  
핵심은 새로운 작업을 시작하는 것이 아니라, **현재 작업 상태를 유지한 채 짧은 정보성 질문만 처리하고 곧바로 복귀**하는 데 있습니다.

# 한눈에 보기(요약)

### 역할

- 현재 작업 상태를 잠깐 고정
- 질문에 대해 짧고 직접적으로 답변
- 필요 시 현재 파일/코드를 읽어 설명
- 답변 직후 원래 작업으로 복귀

### 언제 쓰나

- 작업 중 갑자기 코드 의도나 에러 의미가 궁금할 때
- 지금 수정 중인 코드에 대해 짧은 설명이 필요할 때
- 현재 방향을 바꾸지 않고 개념만 확인하고 싶을 때
- 메인 작업 흐름을 끊지 않고 사이드 질문만 처리하고 싶을 때

### 출력(결과물)

- `ASIDE:` 형식의 짧은 답변
- 필요 시 현재 접근 방식의 위험 신호
- 마지막 줄의 `Back to task` 복귀 안내

# 어떻게 쓰나(1분 사용법)

1. `/aside <질문>` 형태로 입력합니다.
2. 에이전트가 현재 작업 상태를 유지한 채 질문에만 답합니다.
3. 답변이 끝나면 원래 작업으로 바로 복귀합니다.

예:

`/aside 이 함수는 실제로 무엇을 반환해?`

# 이 커맨드가 하는 일(진행 흐름)

### 1) 현재 작업 상태 고정

- 지금 어떤 작업을 하고 있었는지
- 어느 단계에서 멈췄는지
- 다음에 무엇을 할 예정이었는지

이 세 가지를 먼저 머릿속에 고정합니다.

### 2) 질문에 직접 답변

- 결론부터 짧게 답합니다
- 필요하면 현재 파일이나 코드 위치를 읽어 근거를 확인합니다
- 이 과정에서는 파일을 절대 수정하지 않습니다

### 3) 예외 상황 처리

- 질문이 사실상 방향 변경이면, 정보성 질문인지 작업 변경인지 다시 확인합니다
- 답변 과정에서 현재 접근의 위험이 드러나면, 그 사실을 먼저 경고합니다
- 질문이 너무 모호하면 한 번만 짧게 되묻습니다

### 4) 원래 작업으로 복귀

- 답변 후에는 원래 작업을 다시 이어갑니다
- 특별한 blocker가 드러난 경우에만 사용자 결정을 기다립니다

# 실무 팁 / 주의사항(중요)

- `/aside`는 **새 작업 시작용 커맨드가 아니라 잠깐의 우회 질문 처리용**입니다
- 답변 중에는 파일을 절대 수정하지 않습니다
- 질문이 길어질 것 같으면 짧은 답만 먼저 주고, 본 작업 이후에 자세한 설명으로 넘기는 편이 좋습니다
- 답변 때문에 현재 작업 방향을 바꿔야 할 정도면, 그대로 진행하지 말고 먼저 확인을 받는 것이 안전합니다

# Quick Commands(실행 예)

```text
/aside 이 함수는 실제로 무엇을 반환해?
/aside 지금 방식이 thread-safe 한가?
/aside 왜 X 대신 Y를 쓰고 있어?
/aside 방금 추가한 쿼리에 N+1 문제가 있나?
```
