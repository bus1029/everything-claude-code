# `/quality-gate` — 특정 경로 또는 프로젝트 전체에 “온디맨드 품질 체크”를 거는 커맨드

`/quality-gate`는 파일이나 디렉터리를 대상으로 **포맷터, 린트, 타입 체크**를 순서대로 실행해 현재 품질 상태를 점검하는 슬래시 커맨드입니다.  
핵심은 hook이나 CI를 기다리지 않고, **지금 이 시점에 필요한 범위만 골라 품질 파이프라인을 수동 실행**할 수 있다는 점입니다.

# 한눈에 보기(요약)

### 역할

- 대상 경로의 언어/도구 체인을 감지
- formatter 체크 실행
- 가능한 경우 lint/type check 실행
- 수정 가능한 경우 `--fix`로 자동 수정 허용
- 결과를 간단한 remediation list로 정리

### 언제 쓰나

- 커밋 전에 변경 범위를 빠르게 점검하고 싶을 때
- 리팩터링 직후 “최소 품질 게이트”를 수동으로 돌리고 싶을 때
- 전체 프로젝트가 아니라 특정 디렉터리만 확인하고 싶을 때
- CI를 돌리기 전에 로컬에서 먼저 문제를 줄이고 싶을 때

### 출력(결과물)

- 감지된 대상 경로와 사용 도구
- formatter/lint/type check 결과
- 자동 수정 여부
- 남은 이슈와 권장 다음 액션

# 어떻게 쓰나(1분 사용법)

1. 기본값은 현재 디렉터리(`.`)입니다.
2. 특정 경로만 보려면 `/quality-gate src/foo`처럼 경로를 붙입니다.
3. 자동 수정을 허용하려면 `--fix`, 경고도 실패로 보고 싶으면 `--strict`를 붙입니다.

사용 형식:

`/quality-gate [path|.] [--fix] [--strict]`

# 이 커맨드가 하는 일(진행 흐름)

### 1) 대상 범위 결정

- 기본 대상은 현재 디렉터리(`.`)
- 파일 또는 하위 디렉터리를 직접 지정할 수 있습니다

### 2) 도구 체인 감지

- 대상 경로를 기준으로 언어와 설정 파일을 확인합니다
- 해당 언어에서 사용 가능한 formatter, linter, type checker를 선택합니다

### 3) 품질 체크 실행

아래 순서로 진행합니다.

1. formatter 체크
2. lint 체크
3. type check
4. 결과 요약

### 4) 옵션 처리

- `--fix`: 자동 수정 가능한 항목은 수정 허용
- `--strict`: 경고도 실패로 취급 가능한 도구에서는 더 엄격하게 판단

# 실무 팁 / 주의사항(중요)

- 큰 저장소에서는 전체보다 **변경한 경로만 좁혀서 돌리는 편이 빠르고 실용적**입니다
- `--fix`는 반복적인 포맷/간단 린트 수정에 유용하지만, 논리 수정까지 기대하면 안 됩니다
- hook과 CI를 대체한다기보다, **그 전에 빠르게 실패를 줄이는 예비 점검**으로 보는 것이 좋습니다

# Quick Commands(실행 예)

```text
/quality-gate
/quality-gate src
/quality-gate packages/api --fix
/quality-gate . --strict
```
