# `/update-codemaps` — 코드베이스 구조를 스캔해 “토큰-린(요약형) 아키텍처 문서”를 생성/갱신하는 커맨드

`/update-codemaps`는 프로젝트 구조를 분석해 **AI 컨텍스트에 적합한(토큰-린) 코드맵 문서**를 생성/갱신하는 커맨드입니다.  
핵심은 구현 디테일이 아니라 **경계/흐름/핵심 파일과 엔트리포인트**를 짧고 구조적으로 남기는 것입니다.

# 한눈에 보기(요약)

### 역할

- 프로젝트 타입(모노레포/단일 앱/라이브러리 등)과 소스 디렉터리를 스캔
- 엔트리포인트를 찾고(예: `main.ts`, `index.ts`, `app.py`, `main.go`)
- 아키텍처/백엔드/프론트/데이터/의존성 코드맵을 생성/업데이트
- 기존 코드맵이 있으면 변경률(diff %)을 계산해 **대규모 변경 시 승인 게이트** 적용
- 각 코드맵에 생성 메타데이터(생성일/스캔 파일 수/토큰 추정)를 추가
- 변화 요약 리포트를 별도 파일로 저장

### 언제 쓰나

- 큰 기능 추가/리팩터링 이후 “현재 구조”를 문서로 고정하고 싶을 때
- 팀 온보딩/리뷰를 위해 **큰 그림(경계/흐름)**을 빠르게 공유해야 할 때
- AI에게 코드베이스 컨텍스트를 효율적으로 제공하고 싶을 때

### 출력(결과물)

- 코드맵 파일들(예: `architecture.md`, `backend.md`, `frontend.md`, `data.md`, `dependencies.md`)
- 각 문서 상단의 freshness 메타데이터 헤더
- 코드맵 변경 요약 리포트(`codemap-diff.txt`)

# 어떻게 쓰나(1분 사용법)

1. `/update-codemaps`를 실행합니다.
2. 프로젝트 구조/엔트리포인트를 스캔합니다.
3. 코드맵을 생성/갱신하고, 변경률이 큰 경우엔 덮어쓰기 전에 승인 요청을 합니다.
4. 변경 요약 리포트를 저장합니다.

# 이 커맨드가 하는 일(진행 흐름)

### 1) 프로젝트 구조 스캔

- 프로젝트 형태 식별(모노레포/단일 앱/라이브러리/서비스 등)
- 소스 디렉터리 탐색(`src/`, `lib/`, `app/`, `packages/` 등)
- 엔트리포인트 매핑(`main.ts`, `index.ts`, `app.py`, `main.go` 등)

### 2) 코드맵 생성/업데이트

`<CODEMAP_DIR>/` (또는 `<REPORTS_DIR>/codemaps/`) 아래에 아래 문서를 생성/갱신합니다.

- `architecture.md`: 시스템 경계/데이터 플로우/다이어그램
- `backend.md`: API 라우트, 미들웨어 체인, service → repository 매핑
- `frontend.md`: 페이지 트리, 컴포넌트 계층, 상태관리 흐름
- `data.md`: DB 테이블/관계/마이그레이션 히스토리
- `dependencies.md`: 외부 서비스/서드파티/공유 라이브러리

### 3) Diff 감지(승인 게이트)

- 기존 코드맵이 있으면 변경률을 계산
- 변경률이 **30% 초과**면 diff를 보여주고 **덮어쓰기 전 승인 요청**
- 변경률이 **30% 이하**면 인플레이스 업데이트

### 4) 메타데이터(신선도) 헤더 추가

각 코드맵 상단에 다음 같은 헤더를 추가합니다.

```markdown
<!-- Generated: YYYY-MM-DD | Files scanned: N | Token estimate: ~NNN -->
```

### 5) 변화 요약 리포트 저장

`<REPORTS_DIR>/codemap-diff.txt`에 아래 요약을 남깁니다.

- 추가/삭제/수정된 파일
- 새로 감지된 의존성
- 아키텍처 변화(새 라우트/새 서비스 등)
- 90일+ 미갱신 문서에 대한 staleness 경고

# 실무 팁 / 주의사항(중요)

- 구현 디테일 대신 **구조/경계/흐름**을 우선하세요.
- 코드 블록 풀카피보다 **파일 경로/함수 시그니처**가 더 효율적입니다.
- 각 코드맵은 **1000 토큰 이하**를 목표로 유지하는 편이 좋습니다.
- 큰 변경 직후에 생성하면 문서가 “현재 상태”를 가장 잘 담습니다.

# Quick Notes(치환 변수)

- `<CODEMAP_DIR>`: 코드맵 문서 루트(예: `docs/CODEMAPS`, `documentation/codemaps`)
- `<REPORTS_DIR>`: 분석/리포트 산출물 루트(예: `.reports`, `reports`)
- `<APP_DIR>`: 주요 소스 루트(예: `src`, `app`, `lib`, `packages/<name>/src`)
