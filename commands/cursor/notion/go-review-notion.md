# `/go-review` — Go 코드 변경분을 “관용구/동시성/에러처리/보안” 관점에서 리뷰하는 커맨드

`/go-review`는 변경된 Go 코드에 대해 **Go 관용구(idiomatic) 준수**, **동시성 안전성**, **에러 처리 품질**, **보안 취약점(인젝션/레이스 등)**을 중심으로 종합 리뷰 리포트를 생성하는 슬래시 커맨드입니다.  
내부적으로는 **`go-reviewer` 에이전트를 호출**해 Go 전용 체크를 수행합니다.

# 한눈에 보기(요약)

### 역할

- `git diff`로 **변경된 `.go` 파일**을 식별
- `go vet`/`staticcheck`/`golangci-lint` 등 정적 분석 실행
- 인젝션/레이스/고루틴 누수 같은 **보안·동시성 리스크** 점검
- 발견 이슈를 **심각도(CRITICAL/HIGH/MEDIUM)**로 분류해 리포트 생성

### 언제 쓰나

- Go 코드를 작성/수정한 직후
- Go 변경을 커밋하기 직전
- PR에서 Go 변경을 리뷰할 때
- 새 Go 코드베이스에 온보딩하면서 품질 기준을 잡고 싶을 때

### 출력(결과물)

- 리뷰 대상 파일 목록
- 정적 분석 결과 요약(vet/staticcheck/lint 등)
- 이슈 목록(심각도/파일/라인/원인/권장 수정)
- 승인 기준에 따른 결론(Approve/Warning/Block)

# 어떻게 쓰나(1분 사용법)

1. Go 코드를 수정한 뒤 `/go-review`를 실행합니다.
2. 에이전트가 변경 파일과 정적 분석 결과를 기반으로 이슈를 분류합니다.
3. CRITICAL/HIGH 이슈가 있으면 우선 수정하고, 다시 `/go-review`로 재검증합니다.

# 이 커맨드가 하는 일(진행 흐름)

### 1) 변경분 식별

- `git diff`를 기반으로 수정된 `.go` 파일을 찾아 리뷰 범위를 확정합니다.

### 2) 자동 체크(가능하면 실행)

```bash
go vet ./...
staticcheck ./...
golangci-lint run

# 레이스 감지 빌드
go build -race ./...

# 취약점 점검(설치되어 있으면)
govulncheck ./...
```

### 3) 리뷰 관점(핵심)

- **보안**: SQL/command injection, 하드코딩된 시크릿, 위험한 입력 처리
- **동시성**: race condition, 고루틴 누수, 채널 데드락 가능성, 락 범위/패턴
- **에러 처리**: 에러 무시, 맥락 없는 에러 반환, panic 남용, context 전파 누락
- **관용구**: Go 스타일/컨벤션/성능 관점(필요 시)

### 4) 심각도 분류(예시 기준)

- **CRITICAL (반드시 수정)**: 인젝션 취약점, 레이스, 고루틴 누수, 하드코딩 크리덴셜, 중요한 에러 무시 등
- **HIGH (가능하면 수정)**: 에러 컨텍스트 누락, context 미전파, 데드락 위험 채널 사용, mutex 누락 등
- **MEDIUM (검토)**: 비관용구 패턴, export 주석 누락, 미세 성능 개선(프리얼로케이션 등)

# 실무 팁 / 주의사항(중요)

- 병합/배포 안전성을 위해 **CRITICAL/HIGH는 “수정 전까지 Block”**으로 두는 편이 좋습니다.
- 레이스/동시성 이슈는 재현이 어려워도 리스크가 크니, `-race`를 적극 활용하세요.
- `/go-review`는 Go 특화 리뷰이고, 범용 관점은 `/code-review`로 보완하는 구성이 자연스럽습니다.

# Quick Commands(실행 예)

```bash

go build -race ./...
govulncheck ./...
```
